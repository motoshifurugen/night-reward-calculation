# 00 — ハンズオンの始め方

> このガイドは「見本コードを参考に、自分の手でゼロから実装する」ためのハンズオンです。
> コードを写すのではなく、タスクを読んで自分で考えながら書くことを目的としています。

---

## ハンズオンの流れ

```
1. 練習用ブランチを作る
2. タスクガイドを読む（02, 03）
3. 自分でコードを書く
4. テストで確認する
5. 詰まったらヒントを開く
6. 見本コードと答え合わせする
```

---

## Step 1: 練習用ブランチを作る

見本コードとは別のブランチで作業します。

```bash
# main ブランチから練習用ブランチを作成
git checkout main
git checkout -b practice/my-first-react-app
```

---

## Step 2: 見本コードを「答え」として手元に置く

見本コードは `feat/21-initial-setup-top-page-dashboard` ブランチにあります。
別ターミナルで確認できるようにしておくと便利です。

```bash
# 別ターミナルで見本ブランチを確認する場合
git show feat/21-initial-setup-top-page-dashboard:src/components/landing/HeroSection.tsx
```

または GitHub の PR #24 でも確認できます。

---

## Step 3: 環境を確認する

タスクを始める前に、テストが動くことを確認してください。

```bash
pnpm install   # パッケージが入っていない場合
pnpm test      # → テストが実行されることを確認（0 件でも OK）
```

---

## タスクガイドの読み方

各チャプターは以下の構成になっています。

```
## このチャプターで作るもの   ← 完成イメージを確認する
## 事前知識                   ← 必要な概念を読む
## タスク                     ← 自分でコードを書く
  ▼ ヒント（折りたたみ）      ← 詰まったら開く
## 答え合わせ                 ← 見本コードと比べる
## 理解チェック               ← 自分で説明できるか確認する
```

**ヒントは最初から開かないようにしましょう。**
自分で考えてから開くと、より深く理解できます。

## テストの使い方

このハンズオンでは TDD（テスト駆動開発）を使います。

```bash
# テストをウォッチモードで起動（ファイルを保存するたびに自動実行）
pnpm test:watch
```

タスクごとに「テストを先に書いてから実装する」ことを意識してください。

```
テストを書く → 実行して RED を確認 → 実装する → GREEN になる
```
